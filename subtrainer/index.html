IPv4 // Netzwerktechnik

Subnetting Trainer.

Grundrechnen verstehen, nicht auswendig lernen. Leicht-Modus erklärt jeden Schritt einzeln, inklusive Binär. Schwer-Modus wirft dir ungerade Präfixe mitten im Oktett vor die Füße.

Aufgabe Richtig: 0 Serie: 0

Berechne für das folgende Netz die vier Kennwerte:

—
Subnetze bilden Richtig: 0 Serie: 0

Teile das folgende Netz in gleich große Subnetze auf:

—

Trag für jedes Subnetz Netzadresse, ersten und letzten Host sowie Broadcast ein:

VLSM — variable Subnetzgrößen Richtig: 0 Serie: 0

Teile das Netz per VLSM auf. Jeder Standort braucht eine andere Hostzahl — größter Bedarf zuerst, lückenlos vergeben.

—

Trag pro Anforderung Netzadresse, Präfix und Broadcast ein. Reihenfolge: größter Hostbedarf zuerst.

Eine gemischte Aufgabe wie in der Klausur — zufällig generiert, keine Hilfestellung sichtbar, bis du auf Lösung gehst. Wähl die Stufe und arbeite sie sauber durch.

Prüfungsaufgabe Bestanden: 0
Aufgaben werden zufällig berechnet — keine Wiederholung garantiert. Eingaben werden lokal geprüft.
IPv6 // Netzwerktechnik

Subnetting Trainer.

Gleiches Prinzip wie bei IPv4, nur in Hex und 128 Bit. Leicht-Modus erklärt jeden Schritt einzeln, inklusive Hextet-Aufschlüsselung. Schwer-Modus subnettet mitten in eine Hex-Gruppe — wo das Nibble bricht.

Aufgabe Richtig: 0 Serie: 0

Berechne für die folgende IPv6-Adresse die Kennwerte:

—
Subnetze bilden Richtig: 0 Serie: 0

Teile das folgende Präfix in gleich große Subnetze auf:

—

Trag für jedes Subnetz die komprimierte Netzadresse ein:

VLSM — variable Präfixgrößen Richtig: 0 Serie: 0

Vergib aus dem Basispräfix unterschiedlich große Subnetze. Jeder Standort fordert eine andere Präfixlänge — größter Block (kleinste Präfixzahl) zuerst, lückenlos.

—

Trag pro Anforderung die Netzadresse (komprimiert) ein. Reihenfolge: größter Block zuerst.

Bei IPv6 borgt man Präfixe meist auf Nibble-Grenzen. Router-Links nutzen oft /127 (RFC 6164) — analog zu /31 bei IPv4, beide Adressen nutzbar.
Prüfungssituation

Eine gemischte IPv6-Aufgabe wie in der Klausur — zufällig generiert, keine Hilfestellung sichtbar, bis du auf Lösung gehst. Wähl die Stufe und arbeite sie sauber durch.

Prüfungsaufgabe Bestanden: 0
Aufgaben werden zufällig berechnet — keine Wiederholung garantiert. Eingaben werden lokal geprüft. IPv6 kennt keine Broadcast-Adresse und kein −2.
